# Amazon Sales Analytics Portfolio Project

**Tools Used:** Excel | MySQL | Python (Pandas, NumPy, Matplotlib) | Power BI | Tableau

**Dataset:** 100,000 synthetic Amazon-style e-commerce transactions (20 columns — orders, customers, products, pricing, payments, logistics)

## Project Overview

Analyzed Amazon-style sales data to uncover revenue trends, customer behavior, discount impact on profitability, and regional/product performance — using one dataset across the full analyst toolchain: Excel for cleaning, SQL for querying, Python for deeper statistical analysis, and Power BI / Tableau for dashboards and storytelling.

## Key Business Questions Answered

- Which product category generates the most revenue?
- What is the monthly revenue trend — are there seasonal peaks?
- Which payment method is used most often?
- What percentage of orders were Cancelled or Returned?
- Which city has the highest order volume?
- Does a higher discount % lead to more cancellations?
- Who are the top customers by lifetime spend, and how much revenue do repeat customers drive?
- How does revenue and order status vary by category, brand, and seller?

## Key Findings (from Python analysis)

- **Top revenue category:** Electronics
- **Most common payment method:** Credit Card (~35% of orders)
- **Highest order volume city:** Charlotte
- **Cancelled/Returned orders:** ~6% of all orders
- **Repeat customers** drive the large majority of total revenue, despite being a minority of the customer base

## Project Description

> Analyzed 100,000 Amazon e-commerce transactions using Excel for data cleaning and pivot analysis, MySQL for business queries and window functions, Python (Pandas/NumPy/Matplotlib) for exploratory analysis and statistical summaries, Power BI for an interactive sales dashboard, and Tableau for visual storytelling. Answered key business questions around revenue trends, discount impact on profit, customer segmentation, and regional performance.
